# OMX 핵심 용어

## `$deep-interview`

모호한 요구사항과 경계를 먼저 정리하는 in-session surface예요.
현재 OMX 기본 루트의 첫 단계예요.

## `$ralplan`

구현 계획과 tradeoff를 승인하는 planning surface예요.
실행 전에 무엇을 할지 잠그는 단계예요.

## `$team`

작업이 충분히 크고 병렬화가 필요할 때 쓰는 coordinated parallel execution surface예요.

## `$ralph`

한 명의 owner가 검증이 끝날 때까지 계속 밀어붙이는 persistent completion loop예요.

## `omx team`

터미널에서 tmux/worktree coordination을 다루는 operator surface예요.
`$team`과 관련은 깊지만 같은 층위는 아니에요.

## 권장 기본 경로

현재 upstream이 가장 권장하는 기본 환경은 **macOS 또는 Linux + Codex CLI**예요.
Windows는 가능하지만 기본 경험이 아니에요.
